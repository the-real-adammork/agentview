01

Overview & Architecture

What we built, how it's split, and the runtime model the production version should follow.
Read first

What this product is

Observatory is a read-only local dashboard for inspecting Codex agent sessions stored under ~/.codex. The prototype demonstrates five views over the data model defined in the ingestion spec: Sessions, Timeline, Agent Graph, Tokens, Diagnostics.

The visual language is intentionally industrial-instrumentation: dense data, monospace readouts, hazard-tape framing, and a single high-saturation accent. The goal is to make a long-running CLI agent feel observable the way a control room is observable — not the way a SaaS dashboard is.

▸ File layout

Observatory/
├── Observatory.html            — host page, React + Babel bootstrap
├── Component Library.html      — design catalog (no React)
├── Engineering Handoff.html    — this document
├── styles.css                  — single stylesheet, all tokens + components

Production note
The prototype is intentionally three files of JSX for direct-editability. For production, port to TypeScript modules (e.g. Vite + React 18), but keep the CSS-variable architecture in styles.css — it's the cheapest way to ship the four palette channels.

│  · sessions/ │    │  · stream tail    │    └──────┬───────┘
│  · tui.log   │    └───────────────────┘           │
└──────────────┘                                     ▼
                                              ┌──────────────┐
                                              │  React app   │
                                              │  · 5 views   │
                                              │  · live tail │
                                              └──────────────┘
  • Primary index · state_5.threads. Drives the session list with no JSONL parsing.
  • Detail loads · rollout JSONL is parsed lazily when a session is opened, and cached by file mtime + size.
  • Live tail · for active sessions, poll the rollout file incrementally by byte offset.
  • Diagnostics · query logs_2.logs by thread_id on demand; never on the index render path.

▸ Behavior SINGLE SOURCE OF TRUTH

The window is applied to the events array itself, not the rendered list — so the entire Timeline view derives from one filtered slice. This guarantees the scrubber rail, the tab badge counts, the duration in the scrubber header, and the event stream stay mutually consistent.

SurfaceWhat changes when a window is active
Scrubber railDots reduce; t0/tEnd recompute against the slice, so dots re-spread across 0–100%
Scrubber headerLabel swaps to LAST {N}H; duration recomputed from windowed span
Kind tabsPer-kind counts recompute from windowed events (e.g. "Tools · 4" not session total)
Event streamRenders the windowed × kind-filtered intersection
Right rail (vitals, tool usage)Not windowed — these are session totals from session.* fields

▸ Reference time

The reference "now" is the timestamp of the last event in the full session, not the wall clock. Rationale: sessions in this product are historical artifacts on disk — using the wall clock would hide everything in any non-live session.

Live mode
When live-tail (P3) ships, switch refNow to Date.now() only when the session is the active tail target. Detect via the ingest worker's tailThread RPC state. Don't make this a user-visible setting.

▸ Implementation

// 1. State — singular, ms-based; 0 means "no window"
const [windowMs, setWindowMs] = useState(0);

// 2. Derive the windowed events ONCE, memoized.
//    Everything downstream (counts, scrubDots, t0/tEnd, filtered) reads from this.
const refNow = allEvents[allEvents.length - 1].ts;
const events = useMemo(
  () => (windowMs ? allEvents.filter(e => refNow - e.ts <= windowMs) : allEvents),
  [allEvents, windowMs, refNow]
);

// 3. UI — segmented chips inside .tl-tabs, margin-left: auto pushes right.
const WINDOWS = [

▸ Responsive

  • At ≤ 960px the .tl-tabs row gains overflow-x: auto; the range chips scroll inline with the kind tabs.

▸ Edge cases

  • Empty window · if a 1H window contains zero events (rare — the last event always sits at refNow), fall back to the full session for t0/tEnd to avoid undefined.ts.
  • Kind tab + window interaction · they multiply. "Tools · 0" is a legitimate state — render the empty list with a kicker, not a hidden tab.
  • URL state · persist window as a query param (?w=1h) when deeplink/share lands in v0.3.
  • Telemetry · v0.1 has no telemetry; do not add window-selection events without an opt-in pass.
A11y
Wrap the four chips in a role="group" aria-label="Time window". Use aria-pressed mirroring data-on. Keyboard: native Tab navigation between chips; Enter/Space activates. Don't override arrow-key behavior — the surrounding tab row is not a tablist.

▸ Field map — Timeline event

Rollout JSONL is line-delimited typed events. The renderer should:

  • Stream-parse the file by line; never JSON.parse the entire buffer.
  • Build an outByCallId map from function_call_output events for inline joining.
  • Group events by turn_id when rendering turn boundaries.
  • Parse function_call.arguments as JSON when possible; surface raw string when it's not.
  • Treat event_msg.payload.type === 'token_count' as the only source for token snapshots.

Build Plan & Ingest

Recommended phasing and the ingest worker contract.
3 phases

▸ Phasing

PhaseScopeExit gate
P1 · Static index Sessions list backed by state_5.threads. No JSONL parsing. Open in $EDITOR action. Catalog renders 500+ rows in < 200ms. Filters work. No timeline yet.
P2 · Detail views Timeline + Agent Graph + Tokens. Lazy JSONL parse on row click. Cache parsed transcripts. Open largest rollout in repo in < 1s on warm cache, < 4s cold.
P3 · Diagnostics + tail logs_2 queries, live tail of current rollout by byte offset, warn-count badges, raw tui.log fallback. Live tail visible < 500ms after a turn writes. Warn-count badge updates without scroll thrash.

▸ Ingest worker

Run as a separate process / worker thread. Responsibilities:

  • Open state_5.sqlite read-only with WAL mode; keep one persistent connection.
  • Watch ~/.codex/sessions/ for new/changed files; debounce 250ms.
  • For each "opened" thread, stream-parse its rollout into a derived JSON cache keyed by (path, mtime, size).
  • Expose a small RPC: listSessions(filter), getThread(id), tailThread(id, fromByte), queryLogs(filter).
  • Never block the UI thread on JSONL reads > 4MB — page-fetch in 256KB chunks.

▸ Cache shape

// ~/.codex/.observatory/cache/<thread_id>.json
{
  version: 1,
  source: { path, mtime, size },
  turns: [{ id, started_at, completed_at, ttft_ms, duration_ms, … }],
  events: [{ ts, kind, …flat fields }],
  tokens: [{ ts, …token_count payload }],
  edges: [{ parent, child, status, type:"spawn|wait" }],
  summary: { total_tokens, failed_tools, warnings }
}

Invalidate when (mtime, size) differ. Append-only files (active rollouts) only re-read the tail past the cached size.

Local-first
All data stays on disk. No network calls. Default the dashboard to read-only; never write back to state_5 or logs_2. Export/sharing is out of scope for v0.1.

Accessibility, Performance & Privacy

Non-visual gates that block ship.
3 gates

▸ Accessibility

  • Color contrast · ink-strong on bg-0 = 11.4:1 (AAA). ink-dim on bg-0 = 5.3:1 (AA large). Never use ink-faint for primary text.
  • Focus rings · all button and tr[role="button"] get a 1px primary outline with 2px offset. Don't rely on hover background.
  • Reduced motion · gate .blink, .pulse, .ticker behind @media (prefers-reduced-motion: no-preference). Scanlines also off by default in that mode.
  • Screen readers · all hazard tape is aria-hidden. Reticle, brackets, status dots are decorative.
  • Tables · use real <table> with scope="col" on headers. Active rows get aria-current="true".

▸ Performance

  • Render budget · Sessions view must paint < 200ms for 500 rows. Use windowed lists if catalog grows past 500.
  • JSONL parse · stream by line, never bulk-parse. Hand off to worker for files > 1MB.
  • SQLite · keep one read-only connection per DB; never open per-query.
  • Reflow · async warn-count and failed-tools must not push column widths around — preallocate min-width on those cells.
  • Live tail · cap to 200 visible log rows; older rows drop off.

▸ Privacy

  • Default-collapse tool output · > 4KB stays collapsed until user expands.
  • Secret masking · scan tool args + output preview for AWS keys, JWTs, common env-var formats. Replace with [REDACTED] in preview only — never alter underlying data.
  • Don't ship base_instructions · hide payload.base_instructions.text by default; offer "show raw" with a confirm.
  • Local only · no telemetry, no export buttons in v0.1. Reads only.
