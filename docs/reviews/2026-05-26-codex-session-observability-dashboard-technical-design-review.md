## Findings

- [Medium] Timeline scrubber is a required composite pattern and QA gate, but the technical design does not assign it to a component, data contract, or rendering flow. Requirement: `Timeline Scrubber` / `Timeline P2 Scrubber shows >= 20 events with correct kind-by-color`. Design location: `none`. Recommendation: add `TimelineScrubber` to `TimelineView` responsibilities, define the event positioning inputs it consumes from `TimelineEvent`, and state that it is non-interactive in v0.1.
- [Medium] Session filtering and failed-tool session metadata are underspecified relative to the required overview behavior. Requirement: `Session Overview useful filters` and handoff `Session Table Row` / `Sessions overview acceptance`. Design location: `Responsibilities > SessionsView`, `Data Contracts > SessionSummary`, `Integration Points > listSessions`. Recommendation: define `SessionFilter` explicitly with repo/cwd, date range, source, agent role, model, archived, warnings, failed tools, token threshold, and search over title, first message, and full id; add `failedToolCountStatus`/`failedToolCount` or state why failed-tool filtering waits until parsed facts are available.
- [Medium] The ingest-worker contract is only partially reconciled with the handoff. Requirement: `Ingest worker` responsibilities: read-only SQLite with WAL mode, sessions watch debounce, worker/process isolation, RPC surface, no UI blocking above 4MB. Design location: `Proposed Architecture`, `Configuration`, `Performance Strategy`. Recommendation: either add these worker responsibilities to the Node ingest/API design or document intentional deviations, especially the absence of a session-file watcher and the 1MB vs 4MB threshold difference.
- [Low] Visual-token and typography constraints are covered through "port the prototype" language, but several hard requirements are not explicit enough for task planning. Requirement: `Design Tokens`, `Typography`, `Top hazard strip + status bar`. Design location: `Responsibilities > src/frontend/styles/*`, `UI`, `Verification Strategy`. Recommendation: add implementation constraints for no raw component hex values, palette-channel overrides, square corners/1px borders, required top/status chrome on every screen, typography role boundaries, and chrome casing/tracking.
- [Low] Accessibility acceptance details are not fully represented as design obligations. Requirement: `Accessibility` gate. Design location: `Verification Strategy`, `UI`. Recommendation: add design-level requirements for contrast targets, focus rings on buttons and clickable rows, `aria-current` for active rows, `scope="col"` headers, and `aria-hidden` treatment for decorative hazard tape, reticles, brackets, and status dots.

## Recommended Design Changes

1. Add a `TimelineScrubber` subsection under `TimelineView` and `Data Contracts`, including source event positions, kind-color mapping, turn-duration header fields, a minimum >=20 event rendering target, and explicit non-interactive v0.1 behavior.
2. Define `SessionFilter` and complete `SessionSummary` with `preview` fallback semantics plus failed-tool status/count handling. Include search over title, first message, and full id; repo/cwd, date range, source, agent role, model, archived, warning, failed-tool, and token-threshold filters.
3. Reconcile the ingest-worker handoff with the chosen Node API: document whether the API is also the worker, whether file watching/debounce is in v0.1, how read-only SQLite/WAL is opened, and which parse threshold is authoritative.
4. Expand the UI/design-system section with hard prototype constraints: CSS-variable colors only, palette-channel overrides, 1px square borders, required top/status chrome on every screen, typography roles, uppercase chrome, and tabular numeric rendering.
5. Add accessibility obligations, not just tests: contrast targets, focus rings for buttons and clickable rows, `scope="col"` table headers, `aria-current` active rows, and `aria-hidden` decorative hazard/reticle/bracket/status-dot elements.
6. Add an implementation note that the handoff's `Open in $EDITOR` P1 action is superseded by the human v0.1 decision and must be implemented only as copy/reveal path text, with no command or editor endpoint.
